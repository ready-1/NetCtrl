# Prompt Analysis Process

## Overview
Every prompt received by the system must undergo exactly three analysis iterations before any implementation begins. This structured approach ensures thorough understanding and consideration of all aspects before taking action.

## Three-Iteration Methodology

### First Iteration: Basic Understanding
- Identify core requirements
- Understand immediate implications
- Recognize basic patterns
- Map to existing system capabilities

### Second Iteration: Deeper Analysis
- Examine process impacts
- Consider edge cases
- Identify potential challenges
- Refine initial understanding
- Connect with existing system patterns

### Third Iteration: Integration & Synthesis
- Synthesize insights from previous iterations
- Consider system-wide implications
- Identify cross-requirement impacts
- Validate against all system rules
- Prepare for implementation phase

## Integration with Other Requirements

The three-iteration analysis process integrates with other critical requirements:

### Validation Requirements
- Each iteration must validate against system state
- Findings must be tested against existing rules
- Iterations inform validation strategy

### Documentation Requirements
- Each iteration must be documented
- Insights must be preserved for future reference
- Documentation must show progression of understanding

### Sequential Change Requirements
- Analysis iterations must complete before changes begin
- Each iteration builds on previous findings
- No implementation during analysis phase

## Critical Rules

### Required Actions
- Perform exactly three analysis iterations for every prompt
- Document insights from each iteration
- Refine understanding with each iteration
- Apply learnings from previous iterations
- Validate against all rules after each iteration

### Forbidden Actions
- Proceed with implementation before completing all iterations
- Skip or combine iterations
- Reuse analysis patterns without refinement
- Start implementation during analysis phase

## Example Process

```
Prompt: "Update system configuration"

Iteration 1:
- Basic understanding of configuration requirements
- Identify affected components
- Map to existing capabilities

Iteration 2:
- Examine impact on system processes
- Consider security implications
- Identify potential risks
- Refine approach based on first iteration

Iteration 3:
- Synthesize complete understanding
- Validate against all system requirements
- Prepare implementation strategy
- Ensure all rules are satisfied
```

## Implementation Guidelines

1. Each iteration must be clearly documented
2. Iterations must show progressive refinement
3. No implementation work during analysis
4. All iterations must complete before proceeding
5. Each iteration must provide new insights
